### The Funnel Conversion

```
Level 1 → Level 2: Supply ÷ Demand Ratio
  5 experts : 5 parents = 1:1 match potential

Level 2 → Level 3: Parent Search Rate
  ~50-70% of parents search = "active parents"
  5 parents × 60% = 3 active searchers

Level 3 → Level 4: Booking Conversion Rate
  ~30-50% of searchers book = "committed parents"
  3 searchers × 40% = 1.2 bookings

Level 4 → Level 5: Completion Rate
  ~80-90% of bookings complete = "successful consultations"
  1.2 bookings × 85% = 1 completed consultation

Level 5 → Level 6: Payment Processing
  ~95%+ payment success = "revenue"
  1 consultation × $100 × 15% = $15 revenue

Level 6 → Level 7: Repeat Rate
  ~20-30% repeat booking = "proven retention"
  1 successful consultation × 25% = 0.25 repeat bookings

WEEK 4 PROJECTION:
25 experts × 75 parents = 1,875 possible matches
1,875 × 60% (searchers) = 1,125 active searches
1,125 × 40% (converters) = 450 bookings
450 × 85% (completion) = 383 consultations
383 × 15% (commission) × $100 (avg rate) = $5,745 revenue
383 × 25% (repeat) = 96 repeat bookings next month
```

## DECISION FILTER: "Will This Increase Parent-to-Expert Outcomes?"

**Every feature request, every decision, every sprint must answer:**

```
┌─────────────────────────────────────────┐
│ "Will this increase successful          │
│  parent-to-expert consultations?"       │
│                                         │
│ YES → Build it (or prioritize it)       │
│ NO  → Backlog it (or kill it)           │
└─────────────────────────────────────────┘
```

### Examples

**Feature Request: "Add messaging system between parent and expert"**
```
Question: "Will this increase successful consultations?"
Answer: YES - Parents can clarify needs, experts can ask questions
Impact: Higher match quality → higher booking rate → more consultations
Decision: BUILD IT (Track A enhancement)
```

**Feature Request: "Add expert ratings/reviews system"**
```
Question: "Will this increase successful consultations?"
Answer: YES - Social proof increases parent confidence in booking
Impact: Higher conversion on expert profiles → more consultations
Decision: BUILD IT (but AFTER Week 1 - add to Week 2)
```

**Feature Request: "Add AI chatbot to answer parent questions"**
```
Question: "Will this increase successful consultations?"
Answer: MAYBE - Could help, but not critical
Impact: Might accelerate parent decisions, but we have bigger gaps
Decision: BACKLOG IT (not essential for core outcome)
```

**Feature Request: "Add mobile app"**
```
Question: "Will this increase successful consultations?"
Answer: EVENTUALLY - Yes, but not immediately
Impact: Only 20% of target market uses mobile-first
Decision: BACKLOG IT (build after web platform proven)
```

**Feature Request: "Add appointment reminders"**
```
Question: "Will this increase successful consultations?"
Answer: YES - Reduces no-shows, ensures consultations happen
Impact: Currently booked appointments that don't happen = waste
Decision: BUILD IT (Track A enhancement, Week 2 priority)
```

**Feature Request: "Rebrand to 'TheSnNetwork' instead of 'CareLink'"**
```
Question: "Will this increase successful consultations?"
Answer: NO - Branding doesn't drive consultations
Impact: Distracting, uses resources that could build features
Decision: KILL IT (or defer indefinitely)
```
