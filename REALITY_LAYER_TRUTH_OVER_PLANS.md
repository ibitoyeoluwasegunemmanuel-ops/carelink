## LEADING vs. LAGGING INDICATORS

### What Matters (Leading Indicators)

Track these DAILY:

```
EXPERT SIDE:
□ Outreach attempts (per day)
□ Response rate (% who reply)
□ Reply quality (expressing interest?)
□ Time to verification (how long?)
□ Verification completion rate (% who complete?)

PARENT SIDE:
□ Outreach attempts (per day)
□ Response rate (% who reply)
□ Signup rate (% who create account)
□ Search rate (% who search after signup)
□ Booking rate (% who attempt booking)

MARKETPLACE:
□ Match generation rate (matches per day)
□ Booking confirmation rate (% of matches that book)
□ Consultation completion rate (% of bookings that complete)

PARTNERSHIP:
□ Org outreach (per day)
□ Response rate (% who reply)
□ Discussion quality (interest level)
□ Close rate (% who commit)
```

### What DOESN'T Matter (Lagging Indicators)

Track these WEEKLY (not daily):

```
AVOID OBSESSING OVER:
❌ Total user signups (after we know signup rate)
❌ Monthly revenue (until we have 20+ consultations)
❌ Growth metrics (until product works)
❌ Funding progress (until we have proof)
❌ Media mentions (until we have real product)
❌ App store ranking (we're not mobile yet)
```

### The Rule

```
LEADING INDICATORS show what's working.
LAGGING INDICATORS show the result.

Lead.
Then lag follows.

If leading indicators are bad:
→ Fix them immediately
→ Lag will take care of itself

If lagging indicators are bad but leading are good:
→ Be patient
→ Lag will improve in 2-4 weeks

If both leading and lagging are bad:
→ Product doesn't work
→ Pivot strategy
```

## THE HIERARCHY OF OUTCOMES

### Tier 1: Proof of Concept (Weeks 1-4)

**Goal: Prove the model works**

```
Can ANY parent find ANY expert and complete a consultation?

Leading Indicators:
- Expert response rate > 20%
- Parent signup rate > 50%
- Search completion rate > 50%
- Booking rate > 30%
- Consultation completion > 70%

Success = First 5 consultations completed

If any indicator fails → Debug and fix immediately
```

### Tier 2: Repeatability (Weeks 5-8)

**Goal: Prove it works multiple times**

```
Can MULTIPLE parents find MULTIPLE experts and complete consultations?

Leading Indicators:
- Expert signup rate > 30% of contacted
- Parent signup rate > 30% of contacted
- Consultation completion rate consistent > 70%
- Expert satisfaction > 8/10
- Parent satisfaction > 8/10

Success = 20+ consultations completed

If consistency drops → Something broke, find it
```

### Tier 3: Scalability (Weeks 9-12)

**Goal: Prove it scales with volume**

```
Can this work with 50+ experts and 100+ parents?

Leading Indicators:
- Ratio of experts to parents sustainable (1:5 or better)
- Consultation booking time remains < 3 days
- Completion rate stays > 70%
- Repeat booking rate > 15%
- Revenue per consultation stable

Success = 100+ consultations completed

If indicators degrade → System is breaking at scale, redesign
```

### Tier 4: Business Viability (Week 13+)

**Goal: Prove this is a sustainable business**

```
Can this generate predictable revenue and repeat usage?

Leading Indicators:
- Repeat booking rate > 30%
- Expert earnings match promises (are they happy?)
- Parent satisfaction > 8/10
- Partnership referral rate > 20%
- Organic growth rate > 10% MoM

Success = $5K+ MRR with 50%+ repeat usage

If business metrics fail → Unit economics broken, pivot model
```

## WHEN TO PIVOT

### You Should Pivot If (Reality Shows):

```
🔴 Expert Response Rate < 10% for 2+ weeks
   → Your positioning/messaging is wrong
   → Try different narrative
   
🔴 Parent Response Rate < 10% for 2+ weeks
   → Your acquisition channel is wrong
   → Try different channel/messaging
   
🔴 Parent Abandonment at Search > 80% for 2+ weeks
   → Your UX is broken
   → Redesign search experience
   
🔴 Booking Rate < 10% of searchers for 2+ weeks
   → Expert profiles lack credibility
   → Add more profile details/social proof
   
🔴 Consultation Completion < 40% for 2+ weeks
   → Matching is bad or expectations misaligned
   → Improve matching, set clearer expectations
   
🔴 Expert Satisfaction < 6/10 for 2+ weeks
   → Model isn't working for experts
   → Redesign expert experience/commission structure
```

### What NOT to Pivot On

```
❌ "Week 1 we're behind schedule"
   → Not a pivot signal, just execution lag
   
❌ "We only got 1 consultation in Week 2"
   → Expected, give it 4 weeks to assess pattern
   
❌ "We didn't hit all 5 partnerships"
   → Expected, partnerships take time
   
❌ "One expert said they're unhappy"
   → Listen, but need pattern, not outlier
   
❌ "Mobile users requested an app"
   → Nice to have, wait for Volume 2
```
