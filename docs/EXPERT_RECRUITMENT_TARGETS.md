## Verification Process

### Step 1: Document Verification (2-3 days)

```
Review submitted documents:
├─ License validity & status
├─ Issuing body verification (check databases if available)
├─ Educational credentials
├─ No obvious forgery/tampering
└─ Pass/Fail: Missing docs = request within 2 days

Approx time: 15 minutes per expert
```

### Step 2: Background Check (2-5 days)

```
Online verification:
├─ Google search: Name + profession + location
├─ Check professional association listings
├─ Verify current employment (call clinic/hospital)
├─ No negative news/complaints (ethical issues)
└─ Create professional profile background

Approx time: 10-15 minutes per expert
```

### Step 3: Reference Verification (3-5 days)

```
Contact submitted references:
├─ Call/Email 2 professional references
├─ Ask: "How long have you known this person?"
├─ Ask: "Would you recommend them for professional network?"
├─ Ask: "Any concerns about working with families?"
└─ Document responses

Approx time: 5-10 minutes per reference call (2 per expert)
```

### Step 4: Final Review & Approval (1-2 days)

```
Expert Acquisition Lead review:
├─ Summary: All documents + background + references
├─ Decision: APPROVE / REQUEST MORE INFO / REJECT
├─ APPROVE: Expert invited to onboard
├─ REQUEST: Contact expert for missing info
├─ REJECT: Polite decline (rare, ~5-10%)
└─ Document decision

Approx time: 5 minutes per expert decision
```

### Total Verification Timeline: 7-10 days per expert

```
Day 1: Documents submitted
Day 2-3: Document review
Day 4-5: Background check, reference outreach
Day 6-8: Reference responses, compilation
Day 9: Final review
Day 10: Approval/Decision communicated
```

## Risk Management

### Risk 1: Application Dropouts

```
Challenge: Experts apply but don't complete verification
├─ Expected rate: 30% dropout
├─ Mitigation: Warm follow-up emails/calls
├─ Mitigation: Make it easy (simple forms, clear docs)
├─ Mitigation: Show social proof (X experts already approved)
└─ Plan: Keep 150 applications to get 100 verified
```

### Risk 2: Specialization Imbalance

```
Challenge: Too many of one type, too few of another
├─ Target: Balanced mix per specialization
├─ Early warning: Track weekly by specialization
├─ Mitigation: Pivot recruitment to underrepresented types
├─ Contingency: Accept 80-20 ratio (e.g., 25 speech vs 15 OT)
└─ Outcome: Launch with what we have (quality > perfect numbers)
```

### Risk 3: Geographic Concentration

```
Challenge: Too many Lagos, too few outside Lagos
├─ Target: 30% Lagos, 20% Abuja, 50% other cities
├─ Early warning: Track geographically
├─ Mitigation: Partner with NGOs outside major cities
├─ Mitigation: University outreach in secondary cities
├─ Contingency: 60-40 split acceptable (not ideal)
└─ Outcome: Still good geographic representation
```

### Risk 4: Credential Verification Delays

```
Challenge: Can't reach references, slow document verification
├─ Prevention: Good application process (clear docs)
├─ Mitigation: Schedule reference calls during business hours
├─ Mitigation: Have backup references if primary doesn't respond
├─ Contingency: Accept conditional approval (pending one reference)
└─ Timeline buffer: Start verification immediately after application
```

### Risk 5: Expert Attrition During Week 8

```
Challenge: Verified experts go quiet before launch
├─ Prevention: Regular communication (expert group)
├─ Mitigation: Final confirmation calls week 7
├─ Mitigation: Highlight launch momentum to keep momentum
├─ Contingency: Prioritize confirmed vs. non-responsive
└─ Plan: Maintain 100 confirmed by launch day
```
