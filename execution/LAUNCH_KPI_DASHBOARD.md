# Launch KPI Dashboard & Metrics Framework

**Purpose**: Track execution progress toward 4 core success metrics  
**Tool**: Google Sheets dashboard with daily/weekly updates  
**Owner**: Operations Lead / Founder  
**Measure**: Execution velocity, not perfection  

---

## Core Success Metrics

### Metric 1: Expert Signups & Verification

**What**: 100 verified experts on platform by Week 8  
**Why**: Credibility signal. Empty expert directory = dead platform.  
**How to Track**: Expert Recruitment CRM

```
Target Timeline:
├─ Week 3: 30 applications, 20 verified
├─ Week 4: 60 applications, 40 verified
├─ Week 5: 80 applications, 60 verified
├─ Week 6: 100+ applications, 80 verified
├─ Week 7: 100+ applications, 100 verified
└─ Week 8: 100 verified + Active

Dashboard Fields:
├─ Total applications this week: [#]
├─ Total verified this week: [#]
├─ Cumulative applications: [#/target]
├─ Cumulative verified: [#/100]
├─ Applications → Verified conversion rate: [%]
├─ Days average to verify: [#]
└─ On pace: Yes/No (Red/Yellow/Green)
```

**Daily Update**: Pull count from Expert Recruitment CRM  
**Owner**: Expert Acquisition Lead  
**Definition of Verified**: 
- ✓ All documents received
- ✓ Credential verified
- ✓ Background check passed
- ✓ References verified
- ✓ Final approval given

├─ Week 6: 70 signups (organic growth)
├─ Week 7: 85 signups (pre-launch momentum)
└─ Week 8: 100 signups (launch day)

├─ Signup → Login rate: [%]
├─ Average daily active users: [#]
├─ Repeat login rate: [%]

**Owner**: Community Manager  
**Definition of Early User**:

### Metric 3: Active Community Discussions

**What**: 50 active discussions with 1,000+ total posts by Week 8  
**Why**: Activity signals platform is alive. Engagement drives word-of-mouth.  
**How to Track**: Forum analytics + Community engagement tracker

```
Target Timeline:
├─ Week 5: 20 discussions seeded (beta phase)
├─ Week 6: 35 discussions (more topics, parent engagement)
├─ Week 7: 50 discussions (all seeded, organic growth)
└─ Week 8: 50+ discussions with 1,000+ posts (launch momentum)

Dashboard Fields:
├─ Total discussions created: [#/50]
├─ Total posts/comments: [#/1000]
├─ Average posts per discussion: [#]
├─ Parent-authored posts: [#]
├─ Expert responses: [#]
├─ Days since last new discussion: [#]
├─ Most active discussion: [Title], [# posts]
└─ On pace: Yes/No
```

**Daily Update**: Check forum analytics every morning  
**Owner**: Community Manager  
**Definition of Active**: 
- ✓ Discussion created
- ✓ Initial post/prompt written
- ✓ At least 2 responses (can be seeded initially)
- ✓ Mix of parent voices

## Status Color Codes

### Traffic Light System

```
🟢 GREEN = On Track
├─ Metric at or above target
├─ Pace suggests on-time completion
└─ No action required (maintain momentum)

🟡 YELLOW = At Risk
├─ Metric 75-90% of target
├─ Pace slightly behind timeline
├─ May still recover with focused effort
└─ Action: Allocate resources to accelerate

🔴 RED = Behind
├─ Metric <75% of target
├─ Pace suggests missing launch goal
├─ Significant recovery effort needed
└─ Action: Escalate, adjust scope, add resources
```

## Red Metric Recovery Protocol

### If Any Core Metric Turns RED

**Immediate Actions (Within 24 Hours)**

```
Step 1: Diagnose
├─ What's the root cause?
├─ When did it start falling behind?
├─ What changed?
└─ Document in notes

Step 2: Escalate
├─ Notify owner and founder immediately
├─ Explain severity and timeline impact
├─ Propose 2-3 solutions
└─ Get decision on how to proceed

Step 3: Execute
├─ Allocate additional resources
├─ Adjust timeline/scope if needed
├─ Focus team on recovery
└─ Daily updates to leadership

Step 4: Recover
├─ Intensive push to catch up
├─ Daily progress tracking
├─ Escalate second time if not improving
└─ Document lessons learned
```

## Dashboard Implementation (Google Sheets)

### Simple 3-Tab Setup

**Tab 1: Daily Metrics**
```
Columns: Date | Expert #/100 | Parent #/100 | Discussion #/50 | Partnership #/15
Rows: One row per day, Weeks 2-8
Use conditional formatting to color-code: Green/Yellow/Red
Auto-calculate: % of target, pace assessment
```

**Tab 2: Weekly Summary**
```
One row per week
Show: Week #, Each metric final count, Status colors, Notes, Wins, Risks
Summarize key achievements and blockers
```

**Tab 3: Phase Timeline**
```
Timeline view: Week 2-8, each week
Track: When each phase/milestone should complete
Monitor actual vs. plan
Highlight delays immediately
```

**This framework measures execution, not documentation. Success = 4 metrics at target by Week 8.**
